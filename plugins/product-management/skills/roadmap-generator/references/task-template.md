| 유형 | 설명 |
|------|------|

| `backend` | 비즈니스 로직, DB 스키마, API 구현 |

| `test` | 단위 테스트, 통합 테스트, E2E 테스트 |
| `infra` | 배포 파이프라인, 환경 설정, 모니터링 |
| `docs` | API 문서, 사용자 가이드, 릴리스 노트 |

| 사이즈 | 기간 | 기준 |

## 3. RICE 우선순위 테이블

> **RICE 점수** = (Reach × Impact × Confidence) ÷ Effort
>
> - **Reach**: 분기당 영향받는 사용자 수
> - **Impact**: 0.25(최소) / 0.5(낮음) / 1(중간) / 2(높음) / 3(대규모)
> - **Confidence**: 추정 신뢰도 — 50%(낮음) / 80%(중간) / 100%(높음)
> - **Effort**: T-shirt 사이징 기반 person-day (XS=0.5, S=1, M=2, L=4, XL=8)

| ID | 태스크 | Reach | Impact | Confidence | Effort (일) | RICE 점수 | 순위 |

| T-01-02 | {제목} | {N} | 2 | 100% | 2 | {점수} | 1 |
| T-02-02 | {제목} | {N} | 2 | 80% | 4 | {점수} | 2 |

# 릴리즈 노트 생성기 스킬

완료된 태스크와 PRD를 분석해 구조화된 릴리즈 노트를 생성합니다.
단계를 건너뛰지 말고 각 Phase를 순서대로 진행하세요.

---

## 릴리즈 노트 철학

좋은 릴리즈 노트는 세 가지 독자를 동시에 만족시킨다:

- **사용자**: "내가 쓰는 제품이 어떻게 달라졌는가?" — 기능 변화 중심
- **이해관계자**: "BRD에서 약속한 목표가 얼마나 달성되었는가?" — 비즈니스 목표 달성률
- **개발팀**: "무엇이 배포되었고 무엇이 남았는가?" — 기술적 완결성

---

## 대화 프로세스

### Phase 0: 문서 로드

**목표**: 릴리즈 노트 생성에 필요한 소스 파일을 수집한다.

1. 다음 파일을 순서대로 탐색한다:

   ```
   우선순위 1: 사용자가 명시한 경로
   우선순위 2: docs/TASKS-*.md (Glob 검색)
   우선순위 3: docs/PRD-*.md (Glob 검색)
   우선순위 4: docs/BRD-*.md (Glob 검색, 선택사항)
   우선순위 5: git log (Bash로 최근 커밋 조회)
   ```

2. 파일 발견 현황을 사용자에게 보고한다:
   - TASKS.md 없음 → 사용자에게 경로 요청 또는 완료된 기능 목록 입력 요청
   - PRD 없음 → 기능 설명 없이 태스크명만으로 릴리즈 노트 생성 (품질 저하 경고)

3. 릴리즈 범위를 확인한다:
   - "모든 DONE 태스크"인지, 특정 스프린트·버전만인지 물어본다.
   - 버전 번호가 있는 경우 사용자에게 확인 (`v1.0.0`, `v2.3.1` 등)

---

### Phase 1: 완료 태스크 분석

**목표**: TASKS.md에서 DONE 항목을 추출하고 PRD FR과 연결한다.

1. TASKS.md 파싱:
   - `status: DONE` 또는 `[x]` 표시된 모든 태스크 추출
   - 태스크 ID(T-FR-seq), 태스크 유형(design/backend/frontend/test/infra/docs), 크기 추출
   - 연결된 FR-ID 추출

2. FR별 완료율 계산:

   | FR-ID | FR 설명 | 전체 태스크 | 완료 | 완료율 |
   |-------|---------|-----------|------|-------|
   | FR-001 | {설명} | 5 | 5 | 100% ✅ |
   | FR-002 | {설명} | 4 | 3 | 75% ⚠️ |

3. 완전히 완료된 FR(100%)만 릴리즈 노트에 포함한다.
   - 부분 완료 FR은 별도 섹션("다음 릴리즈에 포함 예정")에 기록

---

### Phase 2: BRD 목표 달성률 추적 (BRD 있는 경우)

**목표**: 완료된 기능이 BRD의 비즈니스 목표를 얼마나 달성하는지 보고한다.

```
BRD 목표: "수동 처리 시간 50% 단축 (5일 → 2.5일 이내)"
└── 관련 FR: FR-001 (자동 승인), FR-003 (알림 발송)
    └── 완료 여부: FR-001 ✅, FR-003 ✅
    └── 목표 달성 여부: 측정 필요 (A/B 테스트 또는 운영 지표 확인)
```

달성 여부를 확인할 수 없는 경우 "📊 측정 예정"으로 표시.

---

### Phase 3: git log 분석 (선택)

**목표**: 커밋 이력에서 추가 컨텍스트를 보완한다.

```bash
git log --oneline --since="스프린트 시작일" --until="오늘"
```

커밋 메시지에서 bugfix, hotfix, refactor 등의 키워드를 감지하여 릴리즈 노트의 "버그 수정"/"개선 사항" 섹션에 자동 분류.

---

### Phase 4: 릴리즈 노트 생성 및 저장

**목표**: 사용자·이해관계자·개발팀을 위한 3레이어 릴리즈 노트를 작성한다.

파일명: `docs/RELEASE-NOTES-{버전 또는 날짜}.md`

## 출력 원칙

- **비기술적 언어**: 신규 기능 설명은 개발 용어 없이 사용자 관점으로 작성한다.
- **추적성 유지**: 모든 기능은 FR-ID와 (있으면) BRD REQ-ID를 명시한다.
- **부분 완료 투명성**: 완료되지 않은 항목을 숨기지 않는다.
- **측정 예정 명시**: BRD 목표 달성 여부는 섣불리 "달성"으로 표시하지 않는다 — 운영 지표 확인 필요.
- **다음 단계 안내**: 릴리즈 노트 생성 후 `/retro-generator`로 회고를 진행하도록 안내한다.

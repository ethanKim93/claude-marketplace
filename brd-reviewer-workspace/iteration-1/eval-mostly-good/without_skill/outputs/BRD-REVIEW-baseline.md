# BRD 검토 보고서

**대상 문서:** BRD: 온라인 채용 플랫폼 구축 (v1.2)
**검토일:** 2026-03-31
**검토자:** Claude Code (General Capability / Baseline)
**문서 상태:** Review → 조건부 승인 권고

---

## 종합 의견

전반적으로 잘 작성된 BRD입니다. SMART 목표, 기능 요구사항의 REQ-ID 부여, MoSCoW 우선순위 분류, 리스크 등록부 등 핵심 섹션이 충실하게 작성되어 있습니다. 단, 승인 전 반드시 수정이 필요한 **결함(Defect) 2건**과 보완을 권고하는 **개선사항(Improvement) 4건**이 확인되었습니다.

---

## 1. 섹션별 검토 결과

### 1.1 경영진 요약 (Section 1)
- **판정: PASS**
- 핵심 수치(45일→20일, 월 8건→15건, 예산 4억, 기간 9개월)가 명확히 제시됨.
- 경영진이 읽기에 충분한 요약 수준.

### 1.2 사업 배경 (Section 2)
- **판정: PASS**
- As-Is / To-Be 구조로 현황과 목표를 명확하게 기술.
- 지원자 이탈률 30% 수치는 출처(내부 집계 기간/방법)를 주석으로 명시하면 더 좋음.

### 1.3 프로젝트 목표 (Section 3)
- **판정: PASS**
- G-001, G-002, G-003 모두 지표(Metric), 기준값(Baseline), 목표값(Target), 기한(Deadline)이 명시되어 SMART 기준 충족.
- 세 목표의 기한이 모두 2027-01-31로 운영 전환일(M5: 2026-12-31)보다 1개월 뒤임 — 안정화 기간으로 해석 가능하나, 명시적 주석이 없음.

### 1.4 범위 (Section 4)
- **판정: PASS**
- In-Scope / Out-of-Scope 분리가 명확.
- Out-of-Scope 항목에 "(2단계 추진)", "(추후 검토)" 등 향후 처리 방향이 기재되어 이해관계자 기대관리에 도움이 됨.

### 1.5 이해관계자 분석 (Section 5)
- **판정: PASS**
- 이해관계자 속성(영향력, 참여 수준) 및 RACI 매트릭스가 모두 작성됨.
- RACI에서 PM이 '개발(A)' 역할을 맡고 있으나 실제 개발 책임자(이IT)가 R인 점은 조직 맥락에 따라 재검토 필요.

### 1.6 비즈니스 요구사항 (Section 6)
- **판정: PASS (MoSCoW 비율 주의)**
- REQ-001 ~ REQ-010, NFR-P001 ~ NFR-I001 모두 ID 부여됨.
- MoSCoW Must Have 60%(6/10)는 일반 권고 기준(≤50~60%)의 경계값. 현 수준은 허용 범위이나 추가 Must Have 요건 발생 시 재조정 필요.
- Won't Have 항목이 0건: Out-of-Scope와 역할이 중복될 수 있으나 BRD 수준에서는 허용 가능.

### 1.7 제약 & 가정 (Section 7)
- **판정: DEFECT — 가정 테이블 '영향' 컬럼 미완성**
- 가정 테이블에 '틀렸을 때 영향' 컬럼이 존재하나, 첫 번째 항목("SAP HR 시스템 API 연동이 기술적으로 가능하다")의 영향란이 **공백**임.
- SAP 연동은 리스크 R-001("SAP 연동 기술 제약 / 영향도 높음")과 직결되는 가장 중요한 가정임에도 영향이 기술되지 않아 의사결정에 공백이 발생.
- **필수 조치:** "SAP API 연동 불가 시 배치 연동 전환, 실시간 연동 요건(NFR-I001) 달성 불가 — 범위 및 아키텍처 재검토 필요" 수준으로 기재 필요.

### 1.8 리스크 (Section 8)
- **판정: PASS**
- R-001 ~ R-003에 발생가능성, 영향도, 우선순위, 대응방안, 담당자가 모두 기재됨.
- R-003(일정 지연)의 대응방안 "2주 버퍼"가 마일스톤 일정 테이블에 반영되어 있는지 확인 필요.
- 잠재적 추가 리스크로 "핵심 인력 이탈(채용팀 담당자 교체)"을 검토하도록 권고.

### 1.9 일정 & 마일스톤 (Section 9)
- **판정: PASS**
- M1~M5 모두 완료 기준(Definition of Done)과 담당자, 의존성이 명시됨.
- M1 목표일이 2026-03-31(오늘)이므로 BRD 승인이 오늘 완료되어야 M2 일정에 지장이 없음 — 긴급도 높음.

### 1.10 비용-편익 분석 (Section 10)
- **판정: PASS (편익 추정 근거 보완 권고)**
- 초기 비용 4억원 + 연간 운영 1,500만원 구조가 명확.
- ROI 15%, 투자회수기간 7년 수치는 계산 식이 함께 제시되어 검증 가능.
- "채용 기간 단축 효과 2,000만원/년"은 "추정"으로만 표기되어 있어, 산출 근거(예: 우수인재 채용 실패 시 대체 비용 기준) 보완을 권고.

### 1.11 승인 (Section 11)
- **판정: PASS**
- 역할별 서명란이 완비됨. 현재 서명 미완료 상태이나 Review 단계에서는 정상.

### 1.12 용어집 (Section 12)
- **판정: DEFECT — 용어집 항목 부족**
- 현재 UAT, ROI, FTE 3개 항목만 등록되어 있음.
- 문서 내 전문 용어/약어 중 미등록 항목 다수 확인:
  - **SAP** — 시스템 명칭, 독자에 따라 낯선 용어
  - **RACI** — 책임 분류 매트릭스 약어
  - **MoSCoW** — 우선순위 분류 기법 약어
  - **PoC (Proof of Concept)** — 리스크 R-001 대응방안에 사용
  - **AES-256** — NFR-S001에 사용된 암호화 기술 용어
  - **RTO / RPO** — NFR-A002에 사용된 가용성 용어
  - **GDPR** — NFR-S002에 사용된 규제 약어
  - **REST API** — NFR-I001에 사용된 기술 용어
- **필수 조치:** 최소 비즈니스 이해관계자가 이해해야 하는 비기술 약어(RACI, MoSCoW, PoC, RTO/RPO, GDPR)를 즉시 추가 등록.

## 4. 최종 판정

| 판정 | 조건 |
|------|------|
| **조건부 승인 권고** | D-001(가정 테이블 영향 공백) 및 D-002(용어집 보완) 수정 후 재검토 없이 승인 가능 |

D-001과 D-002는 모두 기존 섹션 내 내용 보완으로 구조 변경이 없으며, 담당자(한지영 PM)가 당일 처리 가능한 수준입니다. 수정 완료 후 이해관계자 서명을 진행하시기 바랍니다.
